# Submission Checklist

- Make the GitHub repository public before submission.
- Commit all source code, `Dockerfile`, `docker-compose.yml`, `k8s/`, `terraform/`, and `.github/workflows/`.
- Upload your final demo video directly into `docs/video/` as an `.mp4` file.
- Update the README with the real Docker Hub image name, AWS domain, and deployment screenshots if available.
- Keep the AWS deployment running until your lecturer verifies it, or record the full demo video before tearing it down.
- Send the public GitHub repository link to your lecturer this week for early verification.
